Mental Model

JavaScript is a dynamically typed, garbage-collected, prototype-based language. The core language is standardized as ECMAScript; browsers, Node.js, Deno, and Bun add host APIs such as DOM, networking, files, timers, and streams.

Execution Model & Event Loop

JavaScript usually executes on one main thread. Synchronous code runs first, then microtasks such as promise reactions, then macrotasks such as timers and many host callbacks.

console.log('A');
setTimeout(() => console.log('D'), 0);
Promise.resolve().then(() => console.log('C'));
console.log('B');
// A, B, C, D

Functions, Scope, and Closures

A closure is a function bundled with access to its lexical environment. Closures power private state, callbacks, factories, and many design patterns.

function makeCounter() {
  let count = 0;
  return function increment() {
    count += 1;
    return count;
  };
}
const next = makeCounter();
next(); // 1
next(); // 2

Objects, Prototypes, and Classes

Objects can delegate property lookup to prototypes. Class syntax is mostly cleaner syntax over prototypes, with modern additions like private fields.

class Pipe {
  #pressure = 0;
  constructor(material) { this.material = material; }
  setPressure(psi) { this.#pressure = psi; }
  get safe() { return this.#pressure < 80; }
}
const p = new Pipe('copper');
p.setPressure(60);
p.safe; // true

Modules

ES modules use static import/export, support tree shaking, and allow top-level await in module contexts.

// math.js
export const add = (a, b) => a + b;
export default function square(x) { return x * x; }

// app.js
import square, { add } from './math.js';
console.log(square(add(2, 3)));

Promises and async/await

A Promise represents eventual success or failure. async functions return promises; await pauses that function until fulfillment or rejection.

async function loadUser(id, signal) {
  const res = await fetch(`/api/users/${id}`, { signal });
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json();
}
const controller = new AbortController();
loadUser(42, controller.signal).catch(console.error);

DOM Events Visual

Browser events travel through capture, target, and bubble phases. Event delegation listens higher in the tree and handles matching children.

document.querySelector('form').addEventListener('click', event => {
  if (event.target.matches('button[data-action]')) {
    console.log(event.target.dataset.action);
  }
});

Testing Starter Pack

Test pure functions heavily, integration paths selectively, and UI behavior from the user’s perspective.

import { describe, it, expect } from 'vitest';
function total(items) {
  return items.reduce((sum, item) => sum + item.price, 0);
}
describe('total', () => {
  it('adds item prices', () => {
    expect(total([{ price: 5 }, { price: 7 }])).toBe(12);
  });
});
